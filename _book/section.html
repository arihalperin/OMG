3.2 Preparation for the Field

(This preparation should be done the day before operations begin)

  1. 1.Program STPR (SBE 39 with pressure)

  2. Supplies needed 1. STPR (aka SeaBird SBE39 temperature and pressure recorder) 2. STPR interface cable (4 pin bulkhead to serial cable) 3. Computer with serial port or USB-serial adapter 4. 9v battery (this will usually be an alkaline battery for fly-in operations since lithiums can't be taken on commercial aircraft) 5. New desiccant pack (if opening housing to swap battery)

  3. Setup SeaTerm to communicate with the STPR (SeaBird model SBE39) – this needs to happen if you are working with a new install of SeaTerm. Once you complete this step, you shouldn't need to do it again for a mission if you are working on the same version of the SeaTerm program. 1. Configure&gt;SBE39&gt;Upload Settings, select Upload data “All as a single file”, also de-select “binary upload”

  4. Check the battery voltage 1. Perform a status check of the instrument by typing “ds” and pressing ENTER. Every command entry should be followed by pressing the ENTER key

    1. 1. If the battery voltage reads less than 7v, click the “Disconnect” button in SeaTerm, and install a new 9v lithium battery in the instrument by unscrewing the endcap of the housing with a wrench.
    2. 2. Replace the used desiccant with a new one prior to closing the housing

  5. Set the instrument's onboard date to UTC TIME IN 24HR FORMAT. 1. Lookup current UTC time from the internet or a GPS unit that is communicating with satellites or some other accurate clock. If need be, calculate UTC time from your current local time by finding out your local timezone offset to UTC time. If you have an internet connection:

    1. 1. This site shows current UTC time in 24HR format which is exactly what we want to use, but the site requires java to view it: http://www.time.gov/timezone.cgi?UTC/s/0/java
    2. 2. If you can't see the time in the previous site, use the following link and convert the listed UTC time to 24HR format:
    3. 3. http://www.timeanddate.com/worldclock/fullscreen.html?n=1440
      1. With your UTC time resource showing, set the CURRENT time by typing the following commands:
    4. 1."MMDDYY=mmddyy" (month, day, year) ex: “MMDDYY=060413”
    5. 2.** HHMMSS**=hhmmss" (hour,minutes, seconds) ex: “HHMMSS=180000”
      1. Perform a status check by typing “ds” followed by ENTER and check to see that the status reflects the new time that you programmed

  6. Set sample interval with the command " interval= your sample rate in seconds"

  7. Ex: " interval =10" will (this may already be set) 1. set start date: " startMMDDYY =mmyydd" 2. set start time to one hour before operations begin IN UTC TIME and 24HRS FORMAT: " startHHMMSS =hhmmss" 3. give command to start later, type " startlater" IF YOU DO NOT GIVE THIS COMMAND, THE STPR WILL NEVER START AT THE PROGRAMMED TIME 4. do a status check ( ds ) to double check values you have input

    1. 1. Current UTC time accurate?
    2. 2. Start UTC time accurate?
    3. 3. Sample interval accurate?
    4. 4. “not logging, waiting to start until (time you set)” displayed?
      1. Click “disconnect” in Seaterm, (there will be no indication that it has disconnected, but it has), close the program and unplug the instrument
      2. Re-install 4-pin bulkhead dummy plug on STPR and stage for field ops tomorrow.
    1. Transfer data from STPR to computer (aka “upload” data in SeaTerm speak) - this occurs daily after operations
      1. Open SeaTerm and connect STPR to computer using the bulkhead connector to serial port cable, using a serial to USB adapter if necessary. The two small rubber bumps on the end of the interface cable should be aligned with the thickest pin on the bulkhead connector of the STPR in order to connect properly.
      2. If the instrument is logging, type “stop”
      3. If the instrument has collected data, upload the data from the instrument to the server.
      4. 1. After instrument is connected to the computer and a connection has been established with SeaTerm, click the button “Upload”
      5. 2. Save the data file to appropriate location on the server
      6. 3. When prompted to enter information for a header enter the mission name and the local date: for Example, “Timor 2013, 2013-JUNE-6” (date being the local date.)
      7. 4. Name the file STPR_serialnumber_local date in YYYYMMDD format_: 1. EX: “STPR39343891538_20130604”
      8. Prepare the SPTPR for the next day of operations: see section d above
      9. Note about the STPR onboard memory: For missions that contain fewer than 25 days of operations, the onboard memory of the STPR will not be cleared during the mission as a single data file at the end of the mission is more efficient to process. Assuming the instrument is stopped from logging overnight and the samplerate is 10s or greater, the onboard memory should provide adequate storage. Daily uploads will continually backup previous uploads and prevent loss of data in the event some previous upload failed. If the memory of the unit became full for some reason, such as being left logging for overnight for several days, it is fine to save the onboard data to the computer and then clear the memory of the unit. Once all data from the unit has been safely saved to a hard drive, the unit's memory may be cleared in SeaTerm by typing the command " samplenum =0" followed by the ENTER key.

3.3 In the Field

  1. 1.Prepare each Niskin bottle prior to getting in the water

  2. With clip and monofilament on each endcap, pull open and “lock” Niskin endcaps open

  3. Ensure that air-vent screw is closed

  4. Ensure that valves are closed (pulled away from bottle)

  5. assign a diver to carry the Niskin bottle and the STPR

  6. 2.Reef Level Samples

  7. During descent to the reef, position open Niskin bottle vertically in the water column to allow bubbles to leave the Niskin bottle and to allow water to flow through it. Upon descending to the reef, place the STPR on the reef where the water sample will be collected. (STPR may be attached to Niskin)

  8. Hold the Niskin bottle close (within 6 inches) to the reef, slide the bottle back and forth to ensure that it washes through with reef level water.

  9. Trigger closure of the endcaps by pressing white trigger button.

  10. Perform a “PAM-pull” to signal the small boat to collect a waypoint over the sample site.

  11. Rest the full Niskin bottle on the reef with it's label (ex. “A”) showing. Hold the diver's computer next to the label and take a picture that captures both the Niskin label and the depth on the computer (use macro to get a clear picture if necessary). This serves as a means to determine which bottle contains the reef sample if field staff forget topside in addition to serving as a depth reference for the sample.

  12. 3.“Surface” Samples

  13. Equip free diver with STPR and prepared Niskin bottle.

  14. Lower Niskin bottle and STPR to depth of one meter, slide Niskin back and forth 3 times to ensure that no bubbles are trapped inside the Niskin., then trigger endcap closure at a depth and notify boat crew to collect a GPS point representing that specific sample (DO NOT COLLECT SAMPLES BY LEANING OVER THE BOAT – they won't be deep enough.) Hold the STPR at sample depth for 30s to ensure that the STPR can collect pressure data.

  15. Upon returning to the boat, place Niskin's in a safe place while divers breakdown gear and prepare for processing. Water samples must be processed as soon as possible.

  16. 5.Filling of bottles for DIC/TA samples

  17. Designate roles: bottle filler, mercuric chloride (HgCl2) handler, data recorder. NOTE: saturated mercuric chloride solution is extremely dangerous; use the utmost caution when dealing with the chemical. All personnel working with it are required to wear eye protection. The mercuric chloride handler is also required to where disposable nitrile gloves. For first aid meaures, see APPENDIX III of this document.

  18. Always collect DIC/TA samples before salinity samples; they are more important and the water chemistry of the samples change with time. Remove a bottle and its stopper from the storage tote and insert the tygon tubing to the bottom of the bottle. With the tygon tubing attached to the Niskin bottle dispensing nipple, open the Niskin bottle valve and allow for three flushings of the bottle to occur before stopping the sample collection (ie. start the collection and count how long it takes for the bottle to overflow and then allow that to occur for 2x the required fill time…ie. if the bottle fills in 20 seconds, allow the sample water flow to flush the bottle for 60 seconds). Attention must be given to how the sample water enters the bottle. Care should be given to ensure that smooth water flow into the bottle is maintained and that no bubbles are created during the dispensing of sample. Any bubbles introduced to the sample will alter the pCO2 within the sample water and produce inaccurate DIC results.

  19. After the appropriate flushing time, shut off the Niskin valve to stop the water flow, while at the same time ensuring the tygon tubing doesn't come off the bottom of the sample bottle. Once the flow is shut off, pinch the tubing and in one motion remove it from the bottle. This “pinch and remove” action with the tubing should establish a consistent head-space in all the sample bottles. The meniscus of the sample should be just below the neck of the bottle.

  20. Once the proper head space is established, pipette 200ul of HgCl2 saturated solution into the sample bottle

  21. Use the syringe containing vacuum grease to make 3-5 vertical “stripes” of grease on a clean, dry stopper. Insert the greased stopper until fully seated in the bottle, then twist until the grease completely seals the bottle contents. The vertical stripes of grease allow for gases to escape the bottle neck while the stopper is being inserted. Having the stopper clean/dry ensures that other than sample water isn't introduced into the bottle. Twisting the stopper, once it has been fully seated into the neck of the bottle, ensures a smooth distribution of grease within the sample bottle's neck and an air tight seal.

  22. Use the rubber band and plastic collar to lock down the stopper inside the bottle. Once secured, softly invert the bottle 1-2x to mix the HgCl2 with the water sample. Label the bottle on the shoulder (to reduce the chance of being rubbed off while being stored) and secure the sample bottle in the field container.

  23. Complete data sheet including waypoint name (default), UTC date and time, lat and long, sample name, bottle labels, site name (if at an established site),

  24. 6.Salinity sample collection:

  25. Flush bottle three times

  26. Fill bottle to very top, in order to reduce headspace in the sample bottle and minimize the possibility for sample evaporation, thus changing the salinity of the sample water.

  27. Fill endcap with sample water and affix to bottle securely

3.4 After the Field Day

  1. Transfer labeled bottles to PMEL tote, ensuring that each is labeled above its shoulder so the ink doesn't rub off when inserted into the tight-fitting foam holes.

  2. 2.Download GPS file to computer (each day)

  3. Establish connection with DNRGarmin

  4. Under “Waypoint” menu, select “Download” and list should populate

  5. Delet all non-default-named waypoints; ex navigation points and any point that is not named with a 3 digit number such as “001”

  6. Under “File” menu, select “Save To” then “File” and save it to appropriate folder on server. An example path is:

CruiseData&gt;Oceanography&gt;Positions&gt;Downloads.

The naming convention should be WYYYYMMDDX where “X” is the single letter label sticker affixed to the GPS unit and the date is the LOCAL DATE of the operation, not the UTC DATE. For example, the GPS waypoint file GPS “H” for local date May 6, 2013 would look like this: W20130506G. Save the file with extension “.txt”

  1. 3.Enter metadata for water samples collected that day into Water Sampling Database (called “CTD Database” MS-Access database.

        1. 1. Import the waypoint file
        2. 2. Enter “CTD Casts” which are actually STPR “casts”
        3. 3. Enter Water Samples associated with each of those casts

  3. Store the hard copy data sheet in the storage binder

  4. 5.Download data from STPR

  5. Connect the instrument to the computer with the cable

  6. Open SeaTearm and connect to the instrument

  7. Save file to appropriate folder on server. An example path is: 1. CruiseData&gt;Oceanography&gt;Moorings&gt;Data&gt;TIM&gt;STPR

  8. Save the file using the following naming convention: STPR_serialnumber_localdat_e. Ex STPR39343891538_20130610.

  9. Follow procedure for preparing for the next field day
